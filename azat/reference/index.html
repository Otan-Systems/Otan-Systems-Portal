01

Overview

This Privacy Policy ("Policy") is issued by Otan Systems ("we", "us", "our") and governs the collection, processing, storage, and use of information in connection with Azat — a specialized AI language model designed for backend development and system architecture assistance.

By accessing or using Azat A1, you ("User") confirm that you have read, understood, and agree to the terms described in this Policy. If you do not agree, you must discontinue use of the service immediately.

Core Principle: Otan Systems is committed to transparency. We collect only what is necessary to operate and improve Azat A1, and we do not sell, rent, or trade personal data under any circumstance.

02

Scope of Application

This Policy applies to all interactions with the Azat A1 system, including but not limited to:

  • Direct usage through the Otan Systems platform or interface.

04

How We Use Data

All data collected is used strictly for the following purposes:

  • Service Delivery: Processing your requests and generating model responses within the defined stack.
  • Performance Monitoring: Measuring response latency, token throughput, and model reliability.
  • Security & Abuse Prevention: Detecting and mitigating unauthorized or malicious usage patterns.
  • Product Improvement: Aggregated, anonymized analytics to improve future versions of Azat A1.
  • Communication: Responding to direct inquiries submitted to official Otan Systems channels.

We do not use your data for advertising, profiling, behavioral targeting, or training third-party models. Any model improvement based on interaction data is performed on fully anonymized and aggregated datasets only.

05

Data Storage & Security

Otan Systems applies industry-standard security practices to protect all data associated with Azat A1. Our infrastructure follows strict operational security principles.

5.1 — Infrastructure

  • Services are containerized using Podman 5.8.2, isolating workloads at the process level.
  • Network traffic is proxied and filtered via Nginx 1.24.
  • Authentication between internal services relies on PASETO tokens — no JWT vulnerabilities apply.
  • All inter-service communication is encrypted in transit (TLS 1.3 minimum).

5.2 — Data Isolation

  • Conversation data is not persisted to disk after session termination by default.
  • Logs are stored in append-only, access-controlled storage with automated expiry.
  • Databases (ArangoDB 4.0, DuckDB v1.4 LTS) hosting RAG knowledge are read-only at query time and do not store user interaction history.

5.3 — Breach Response

In the event of a confirmed data breach affecting user data, Otan Systems will notify affected parties via email within 72 hours of confirmed discovery, in accordance with applicable data protection standards.

07

RAG System & Knowledge Base

Azat A1 utilizes a Retrieval-Augmented Generation (RAG) architecture, embedding curated, filtered documentation from official sources of the supported stack (Go, Python, Ruby, Node.js, ArangoDB, DuckDB, and related libraries).

Important: The RAG knowledge base contains only publicly available official documentation. No user-submitted content, proprietary code, or personal data is ever embedded into or used to update the RAG knowledge base.
  • The RAG base is read-only at inference time — no query modifies the knowledge store.
  • Documentation sources are from official project repositories and maintained by Otan Systems engineers manually.
  • The context window of 262,144 tokens is processed ephemerally and not persisted post-session.

15

AI Processing & Infrastructure

Azat A1 relies on a combination of proprietary and third-party AI infrastructure to deliver its capabilities. This section describes the external AI services involved in data processing.

15.1 — Google Vertex AI

Otan Systems uses Google Vertex AI as part of the underlying AI processing pipeline. Specifically, Vertex AI is involved in:

  • Processing of prompts and generation of model responses where applicable.
  • Powering data processing tasks associated with Azat workflows and workloads.
  • Supporting inference infrastructure for high-availability response delivery.

Data processed via Google Vertex AI is governed by Google Cloud's Data Processing Addendum and applicable Google Cloud privacy terms. Otan Systems has configured Vertex AI services under settings designed to minimize data retention by Google.

Data Residency: Prompts processed through Google Vertex AI may be transmitted to Google Cloud infrastructure. Otan Systems has configured regional processing settings where available, but users in jurisdictions with strict data residency requirements should be aware of this cross-border processing.

15.2 — Scope of Third-Party AI Processing

  • Only the content of active requests (prompts) is sent to Vertex AI — no locally stored history or personal profile data is forwarded.
  • Otan Systems does not permit Google to use submitted prompts for training its own models, in accordance with applicable Google Cloud enterprise data terms.
  • Session context is ephemeral and not persisted beyond the active inference window.
